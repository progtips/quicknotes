# Чек-лист ручного тестирования QuickNotes

## Подготовка к тестированию

- [ ] Backend сервер запущен на `http://localhost:4000`
- [ ] База данных настроена и миграции применены
- [ ] Frontend приложение запущено (Expo web или мобильное приложение)
- [ ] Браузер DevTools открыт для просмотра сетевых запросов

---

## 1. Регистрация нового пользователя

### Тест 1.1: Успешная регистрация
- [ ] Открыть экран регистрации
- [ ] Ввести валидный email (например: `test@example.com`)
- [ ] Ввести пароль (минимум 6 символов)
- [ ] Подтвердить пароль
- [ ] Нажать кнопку "Зарегистрироваться"
- [ ] **Ожидаемый результат:** Пользователь зарегистрирован, получен JWT токен, выполнен автоматический вход, переход на экран списка заметок

### Тест 1.2: Регистрация с существующим email
- [ ] Попытаться зарегистрироваться с email, который уже существует
- [ ] **Ожидаемый результат:** Отображается ошибка "Пользователь с таким email уже существует"

### Тест 1.3: Валидация полей
- [ ] Попытаться зарегистрироваться без email
- [ ] **Ожидаемый результат:** Отображается ошибка валидации
- [ ] Попытаться зарегистрироваться с некорректным email форматом
- [ ] **Ожидаемый результат:** Отображается ошибка "Некорректный email"
- [ ] Попытаться зарегистрироваться с паролем менее 6 символов
- [ ] **Ожидаемый результат:** Отображается ошибка "Пароль должен содержать минимум 6 символов"
- [ ] Попытаться зарегистрироваться с несовпадающими паролями
- [ ] **Ожидаемый результат:** Отображается ошибка "Пароли не совпадают"

### Тест 2.1: Успешный вход
- [ ] Открыть экран входа
- [ ] Ввести существующий email
- [ ] Ввести правильный пароль
- [ ] Нажать кнопку "Войти"
- [ ] **Ожидаемый результат:** Пользователь авторизован, получен JWT токен, переход на экран списка заметок

### Тест 2.2: Неверный пароль
- [ ] Ввести существующий email
- [ ] Ввести неверный пароль
- [ ] Нажать кнопку "Войти"
- [ ] **Ожидаемый результат:** Отображается ошибка "Неверный email или пароль"

- [ ] Ввести любой пароль
- [ ] Нажать кнопку "Войти"
- [ ] **Ожидаемый результат:** Отображается ошибка "Неверный email или пароль"

### Тест 2.4: Валидация полей

- [ ] **Ожидаемый результат:** Отображается ошибка "Заполните все поля"

## 3. Создание заметки

### Тест 3.1: Создание новой заметки
- [ ] Войти в систему
- [ ] Нажать кнопку "+" (FAB) для создания заметки
- [ ] Ввести заголовок заметки
- [ ] Ввести содержимое заметки
- [ ] Нажать кнопку "Сохранить"
- [ ] **Ожидаемый результат:** Заметка создана, отображается в списке заметок, переход на экран списка

### Тест 3.2: Создание заметки только с заголовком
- [ ] Создать новую заметку
- [ ] Ввести только заголовок (без содержимого)
- [ ] Нажать "Сохранить"
- [ ] **Ожидаемый результат:** Заметка создана с пустым содержимым

### Тест 3.3: Валидация заголовка
- [ ] Попытаться создать заметку без заголовка
- [ ] **Ожидаемый результат:** Отображается ошибка "Заголовок обязателен"

### Тест 3.4: Проверка отображения в списке
- [ ] Создать несколько заметок
- [ ] Проверить список заметок
- [ ] **Ожидаемый результат:** Все созданные заметки отображаются в списке, отсортированы по дате обновления (новые сверху)

---

## 4. Редактирование заметки

### Тест 4.1: Редактирование существующей заметки
- [ ] Открыть существующую заметку из списка
- [ ] Изменить заголовок
- [ ] Изменить содержимое
- [ ] Нажать кнопку "Сохранить"
- [ ] **Ожидаемый результат:** Заметка обновлена, изменения отображаются в списке и при повторном открытии

### Тест 4.2: Частичное обновление
- [ ] Открыть заметку
- [ ] Изменить только заголовок (без изменения содержимого)
- [ ] Нажать "Сохранить"
- [ ] **Ожидаемый результат:** Обновлен только заголовок, содержимое осталось прежним

### Тест 4.3: Архивирование заметки
- [ ] Открыть существующую заметку
- [ ] Включить переключатель "Архивировать"
- [ ] Нажать "Сохранить"
- [ ] **Ожидаемый результат:** Заметка архивирована, не отображается в основном списке (если не включен фильтр архивных)

### Тест 4.4: Фильтр архивных заметок
- [ ] Архивировать несколько заметок
- [ ] Нажать кнопку "Включая архивные" в списке заметок
- [ ] **Ожидаемый результат:** Архивные заметки отображаются в списке с пометкой "Архив"

---

## 5. Удаление/Архивация заметки

### Тест 5.1: Мягкое удаление заметки
- [ ] Открыть существующую заметку
- [ ] Нажать кнопку "Удалить"
- [ ] Подтвердить удаление в диалоге
- [ ] **Ожидаемый результат:** Заметка удалена (мягкое удаление), не отображается в списке

### Тест 5.2: Отмена удаления
- [ ] Открыть заметку
- [ ] Нажать "Удалить"
- [ ] Нажать "Отмена" в диалоге подтверждения
- [ ] **Ожидаемый результат:** Заметка не удалена, остается открытой

### Тест 5.3: Проверка недоступности удаленной заметки
- [ ] Удалить заметку
- [ ] Попытаться открыть удаленную заметку напрямую (если есть прямой доступ)
- [ ] **Ожидаемый результат:** Заметка не найдена или не отображается

## 6. Синхронизация между устройствами

### Тест 6.1: Вход с другого устройства
- [ ] На первом устройстве: создать несколько заметок
- [ ] На втором устройстве: войти с теми же учетными данными
- [ ] **Ожидаемый результат:** Все заметки с первого устройства отображаются на втором устройстве

### Тест 6.2: Создание заметки на втором устройстве
- [ ] На втором устройстве: создать новую заметку
- [ ] На первом устройстве: обновить список заметок (pull-to-refresh)
- [ ] **Ожидаемый результат:** Заметка, созданная на втором устройстве, появляется на первом

### Тест 6.3: Редактирование на разных устройствах
- [ ] На первом устройстве: открыть заметку
- [ ] На втором устройстве: отредактировать ту же заметку
- [ ] На первом устройстве: обновить список
- [ ] **Ожидаемый результат:** Изменения с второго устройства отображаются на первом

### Тест 6.4: Одновременное редактирование
- [ ] На первом устройстве: начать редактирование заметки
- [ ] На втором устройстве: отредактировать и сохранить ту же заметку
- [ ] На первом устройстве: сохранить изменения
- [ ] **Ожидаемый результат:** Последние изменения сохраняются (возможны конфликты, которые нужно обработать)

### Тест 6.5: Проверка JWT токена
- [ ] Войти на первом устройстве
- [ ] Скопировать JWT токен из localStorage/SecureStore
- [ ] На втором устройстве: использовать тот же токен (если возможно)
- [ ] **Ожидаемый результат:** Доступ к данным с обоих устройств работает корректно
